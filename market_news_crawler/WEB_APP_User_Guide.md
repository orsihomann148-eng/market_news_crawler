# Web App 网页端使用说明

这份文档专门说明 `web_app.py` 的网页端怎么使用，适合日常通过浏览器操作系统。

如果你更关心命令行参数和脚本入口，请继续看 `README.md`。这份文档重点讲：

- 页面从哪里打开
- 每个模块在做什么
- 常见操作应该怎么点
- 页面里保存了哪些设置
- 抓取完成后怎么看结果

## 1. 启动网页端

先安装依赖，即运行代码需要调用的库：

```bash
python3 -m pip install -r requirements.txt
```

启动网页端：

```bash
python3 web_app.py
```

默认访问地址：

```text
http://127.0.0.1:8000
```

打开后你会看到一个 Web 控制台，顶部主要有 5 个标签：

- `总览`
- `新闻抓取`
- `新闻查看`
- `国家管理`
- `来源管理`

## 2. 页面整体逻辑

网页端本质上是把常用脚本包装成了一个浏览器控制台。

当前最重要的主链路是：

```text
web_app.py -> xlsx_source_test.py
```

也就是说：

- 你在网页里点“开始抓取”
- 后端会按当前国家配置、平台选择、搜索关键词和 AI 设置去运行抓取任务
- 跑完后结果会写到 `outputs/` 下
- 你可以直接在“新闻查看”里浏览结果

## 3. 先理解“国家切换”

网页端几乎所有功能都和“当前国家”绑定。

切换国家后，会一起切换：

- 网站库
- 平台列表
- 平台别名
- 搜索地区
- 默认补充检索关键词
- 默认 AI 提示词
- 来源列表
- 输出结果筛选范围
- 国家子目录下的数据文件

国家相关数据文件统一放在：

```text
country_data/<country_code>/
```

例如：

```text
country_data/france/
country_data/germany/
country_data/es/
country_data/italy/
```

## 4. 新闻抓取模块怎么用

`新闻抓取` 是最常用的模块。

### 4.1 基础设置

你先需要确认这些内容：

- `国家切换`
- `时间模式`
- `最近天数` 或 `开始日期 / 结束日期`
- `翻译语言`

说明：

- `最近 N 天` 适合日常增量抓取
- `指定日期范围` 适合回补历史数据
- 翻译语言通常用 `zh-CN` 或 `en`

### 4.2 选择品牌

页面里分成两部分：

- `已有品牌`
- `新增品牌`

#### 已有品牌

这是当前国家配置里已经维护好的平台列表。

例如可能包括：

- `Amazon`
- `TikTok Shop`
- `Zalando`
- `SHEIN`
- `Temu`
- `Instagram`

这些品牌会自动使用该国家对应的搜索别名。

例如法国的 `Amazon`，系统会自动按类似这些词去搜索和匹配：

- `Amazon France`
- `Amazon FR`
- `Amazon.fr`
- `Amazon`

#### 新增品牌

如果你想临时补一个当前国家配置里没有的品牌，可以在“新增品牌”文本框里每行输入一个。

例如：

```text
Mercari
Yahoo! Shopping
```

现在这类手动输入品牌也会自动做轻量别名扩展，不再只搜原词。

例如：

- `IG` 会自动扩展成 `Instagram Shopping / Instagram Shop / Instagram / IG / INS`
- `TTS` 会自动扩展成 `TikTok Shop / TikTok`
- `Amazon FR` 会自动识别回 `Amazon`
- `Mercari` 在法国下会扩展成 `Mercari / Mercari France / Mercari FR`

这能减少只搜原词导致的漏抓。

### 4.3 修改品牌搜索别名

在“已有品牌”下面，有一个折叠区域：

```text
查看和修改品牌搜索别名
```

这里可以直接编辑某个国家下每个平台的搜索别名。

每行格式是：

```text
品牌 | 别名1 | 别名2 | 别名3
```

例如：

```text
Amazon | Amazon France | Amazon FR | Amazon.fr
```

保存后会影响：

- 补充搜索
- 数据/报告搜索
- 新闻结果品牌命中判断
- 页面里的默认别名文本

适合处理这些场景：

- 你发现某个国家某个平台召回偏低
- 某个平台在本地媒体里常用特殊简称
- 想补本地域名，例如 `Amazon.fr`、`Zalando.de`

如果不小心改乱了搜索别名，可以点击恢复默认，然后点保存可以恢复原来的搜索别名。**（可优化）**

### 4.4 选择侧别

侧别有 3 种：

- `media`
- `buyer`
- `seller`

通常建议：

- 日常新闻监测优先勾 `media`
- 如果你也想看平台自己的买家帮助页、卖家中心动态，再加 `buyer / seller`

当前逻辑：

- `media` 会在媒体来源中按已选品牌逐个搜索
- `buyer / seller` 会抓对应品牌自己的来源

### 4.5 补充检索设置

页面里有一块“补充检索任务”。

这里控制两类补充搜索：

- `抓取相关新闻`
- `抓取数据和报告`

这两类内容都来自搜索引擎补充抓取，常用于弥补“固定来源不足”的情况。

#### 搜索引擎

可选：

- `Google + Bing`
- `Google`
- `Bing`

实际使用里，如果你发现某个国家 Bing 质量不稳，可以只用 `Google`。

#### 查看和修改补充检索关键词

这里可以直接改两种关键词块：

- `相关新闻关键词块`
- `数据/报告关键词块`

系统会自动拼成检索语句：

- 相关新闻：`品牌词 + 这一行关键词 + 国家限定词`
- 数据报告：`品牌组 + 这一行关键词 + 国家限定词`

如果你发现某个国家抓到的新闻偏少，通常优先检查：

- 平台别名是否太弱
- 相关新闻关键词块是否太少
- 数据/报告关键词块是否太弱

### 4.6 指标 3.2 筛选方式

网页端支持两种筛选方式：

- `关键词匹配`
- `AI 理解筛选`

#### 关键词匹配

特点：

- 快
- 成本低
- 适合粗筛或无 API 的情况

#### AI 理解筛选

特点：

- 更适合判断新闻是否真的会影响问卷指标 3.2
- 可以区分“平台本身变化”和“只是文章里顺带提到平台”
- 适合最终交付结果

如果用 AI 筛选，你需要填写：

- `AI API URL`
- `AI 模型`
- `AI API Key`

页面支持点击 `测试 API` 先检查连通性。

目前只尝试用过deepseek和kimi的api，deepseek筛选效果会好一些。

### 4.7 修改 AI 系统提示词

在 AI 筛选区域里，有一个折叠区：

```text
查看和修改 AI 提示词
```

这里改的是：

- AI 理解筛选时发送给模型的系统提示词

保存后下次抓取会直接生效。

适合这些情况：

- 某个国家筛选过严，想放宽保留标准
- 某类内容误保留太多，想加强过滤
- 想调整维度映射的倾向

### 4.8 点击开始抓取后会发生什么

点击 `开始抓取` 后：

1. 当前新闻抓取表单会先保存
2. 后端异步创建一个任务
3. 页面右侧会显示任务进度、已选平台数、网站数、剩余时间估算
4. 抓取完成后，结果会写入 `outputs/` 下的国家对应目录

当前进度条已经做了这些优化：

- 不会倒退
- 网站抓取和补充搜索占比已压低
- 翻译、3.2 筛选、AI 去重、写文件分开计时
- 剩余时间更接近实际耗时

## 5. 新闻查看模块怎么用

`新闻查看` 用来直接浏览 `before.csv / after.csv` 等结果文件。为了网站能快速打开，每次打开网站，如果想查看历史抓取结果，需要点击刷新数据文件或切换数据文件，具体的新闻卡片才会显示。

### 5.1 可以筛选什么

页面支持按这些维度筛选：

- `数据文件`
- `平台`
- `关键词`
- `显示条数`
- `是否只看已星标`

### 5.2 数据文件怎么选

页面会自动列出当前国家可用的文章 CSV 文件。

常见会看到：

- `after.csv`
- `before.csv`

建议理解方式：

- `sources.csv`：来源抓取诊断
- `before.csv`：筛选前新闻池
- `after.csv`：最终结果，最适合日常浏览

### 5.3 文章卡片能看到什么

每条新闻卡片通常会展示：

- 品牌标签
- 来源名
- 分类
- 发布时间
- 标题
- 摘要
- 指标映射
- 关联说明
- 原文链接
- 来源页链接

### 5.4 多品牌新闻的展示方式

如果一条新闻同时命中多个品牌，现在网页端会合并展示成一条，而不是拆成多条。

例如：

```text
Amazon France, TEMU
```

这能减少你看到大量“同一网址但品牌不同”的重复项。

### 5.5 星标功能

每条新闻右上角都有 `🌟` 按钮。

你可以用它：

- 标记重要新闻
- 在页面里只筛选已星标新闻
- 导出星标新闻

星标数据保存在：

```text
article_star_store.json
```

会记录哪些新闻已经被你标星
## 6. 国家管理模块怎么用

`国家管理` 主要用来新增国家、让 AI 先帮你生成配置，再保存。

### 6.1 AI 自动填写

使用方式建议是：

1. 先填 `国家显示名`
2. 配好 `AI API URL / API Key / 模型`
3. 点击 `AI 自动填写`
4. 检查生成结果
5. 再点击 `新增国家`

AI 自动填写当前会补这些内容：

- 国家代码
- 国家显示名
- 时区
- 消费者标签
- 市场标签
- 搜索地区参数
- 市场关键词
- 默认品牌列表
- 平台别名
- 官方来源
- 补充检索关键词
- 数据/报告关键词
- 默认文件路径

### 6.2 AI 自动填写的别名规则

这块已经做过强化，不再是“每个平台后面随便加国家名”。

现在它会优先理解品牌语义，例如：

- `IG / INS / INS-DTC` 会按 `Instagram Shopping` 理解
- `TTS` 会按 `TikTok Shop` 理解
- 会补新闻里真实会出现的品牌词、本地域名和国家代码变体

### 6.3 新增国家后会自动创建什么

新增国家保存成功后，会自动创建：

```text
country_data/<country_code>/
```

并准备这些文件：

- `source_survey.xlsx`
- `extra_sources.json`
- `site_adapter_configs.json`
- `site_credentials.json`
- `source_capability_cache.json`

新建国家完成后，需要手动在`source_survey.xlsx`中粘贴当前国家的各指标信息

### 6.4 删除国家

页面里只能删除“自定义国家”，不能删除内置国家（日本，法国）。

删除后会：

- 移除该国家配置
- 清理对应自动生成的国家文件（output中的文件不受影响）
- 清理该国家保存的页面设置

删除前建议先确认：

- 这个国家目录下是否还有你想保留的人工内容
- 是否还有输出结果需要整理

## 7. 来源管理模块怎么用

`来源管理` 用来维护额外来源网站，不是直接看文章结果。

支持的操作：

- `查看列表`
- `新增来源`
- `重新启用来源`
- `停用来源`

### 7.1 新增来源时要填什么

通常需要填：

- 平台名
- 侧别
- 网址

如果是需要自动生成适配器的站点，还要填：

- API URL
- API Key
- API Model

### 7.2 两个常用选项

新增来源时可选：

- `跳过 API 适配器生成`
- `强制重新生成适配器`

适配器的作用是：告诉代码如何在这个新闻网站上搜索新闻

适合场景：

- 你只想先登记网址，不想立刻走 AI
- 某个站点适配规则失效了，想重生一次

### 7.3 查看列表时能看到什么

来源列表会显示：

- 平台
- 侧别
- 网址
- 是否启用
- 完整性风险
- 能力标签
- 最近评估结果

这块很适合排查：

- 某个站点为什么抓不到
- 某个来源是不是已经失效
- 某个来源有没有 RSS / 搜索页 / 公开列表页

## 8. 页面设置会保存什么

网页端会保存不少状态，方便下次打开继续用。

主要包括：

- 上次选择的国家
- 新闻抓取表单状态
- 新闻筛选设置
- 来源管理表单状态
- AI API 设置
- 国家管理 AI 提示词

主要保存在：

```text
web_app_settings.json
```

这意味着：

- 你刷新页面后，很多内容会保留
- 同一个国家会有自己的一套表单状态
- 如果你改了默认值，但没清理当前已保存状态，页面可能继续使用旧值

## 9. 结果文件怎么看

每次主抓取任务完成后，结果会写到 `outputs/` 下。

典型目录名类似：

```text
outputs/france_xlsx_sources_media_buyer_seller_run_20260429_142652/
```

常见文件：

- `metadata.json`
- `sources.csv`
- `before.csv`
- `after.csv`

建议阅读顺序：

1. 看 `sources.csv`
   作用：来源抓取是否成功，失败原因是什么
2. 看 `before.csv`
   作用：补充搜索和来源抓取总共召回了多少新闻
3. 看 `after.csv`
   作用：最终可交付新闻结果

## 10. 日常推荐操作流程

如果你是日常使用网页端，比较推荐这个顺序：

1. 先切到目标国家
2. 检查已选品牌和新增品牌
3. 确认补充检索是否开启
4. 确认筛选方式是否为 `AI 理解筛选`
5. 点 `开始抓取`
6. 等任务完成
7. 去 `新闻查看` 里先看 `after.csv`
8. 如果结果偏少，再回头检查 `before.csv` 和品牌别名
9. 如果站点来源异常，再到 `来源管理` 里排查

## 11. 常见问题

### 11.1 为什么我改了关键词，但任务看起来没变化

常见原因：

- 你只改了文本，没有点击保存
- 页面仍在使用 `web_app_settings.json` 里旧的国家状态
- 你改的是别的国家，不是当前国家

### 11.2 为什么某个国家新闻量突然变少

常见排查顺序：

1. 先看 `before.csv`，判断是召回少还是筛掉多
2. 检查平台别名是否太弱
3. 检查相关新闻关键词块是否太少
4. 检查 AI 提示词是否过严
5. 检查搜索引擎是否只开了 `Bing`

### 11.3 为什么同一条新闻以前会出现多次

之前常见原因是：

- 同一网址命中多个品牌
- 不同媒体在转发同一事件

现在网页端已经做了两层优化：

- 同网址多品牌新闻合并展示
- 最终 AI 去重更强调“同品牌、同事件、跨媒体重复”

### 11.4 为什么页面里能看到旧配置

因为页面会记住当前国家上次保存的状态。即使你改了代码里的默认值，如果该国家已有保存状态，页面也可能继续显示旧内容。

这时要检查：

- `web_app_settings.json`
- 当前国家的筛选设置是否已经重新保存

## 13. 写给后面完善代码的人

每次完成修改后，要ctrl+c退出后重新运行 web_app.py 改动才会生效，尽管codex会告诉你 他已经把代码重启好了。

目前新闻抓取没有强制终止功能，如果抓取途中想修改某些设置，并且当前抓取作废，只能在vscode里ctrl+c退出，网页端会显示fetch failed，后面可以优化。

目前所有的提示词、搜索关键词等网页端可修改的文本，在改完后都需要点击保存才能应用到代码中，恢复默认值点完后也需要点保存才能应用，如果想回到上一次保存内容（非默认值），只能不要点击保存然后刷新网页。默认值也是不能从网页端修改的，以后都可以优化。

目前浏览器网页只作为后台python程序的控制入口，程序运行后也仅限本机进入网站，后续如果需要可以考虑做成脱离后台运行python约束，或者局域网共享网站的版本。

在想把程序打包发给别人时，可以压缩后放心直接全部发送，api key虽然存在本地，但是做了加密处理，程序运行时会自动在user\config文件夹中生成一个.key文件，有这个文件才能正常读取之前存储的api key。

目前输出的csv文件与cs需求中的格式不同，后面可以按需修改。

直接用ai搜索，可能会给你一些相关报告或数据披露的PDF网页，用爬虫很难抓到这些网页。PDF 地址可能是通过 JavaScript、表单提交、API 接口或 CMS 媒体接口动态生成的。普通爬虫只抓\<a href="">，就可能漏掉。难点不是 PDF 本身隐藏得特别深，而是它经常不是静态暴露在 HTML 里的普通链接。GPT/Search 更像“搜索引擎索引 + 页面理解 + 链接追踪”，而普通爬虫如果不执行 JS、不查搜索索引、不分析 API 请求，就很容易漏掉。后面可以优化。
